## Build & Run

After any change to `project.yml`, regenerate the Xcode project before opening in Xcode:
```
xcodegen generate
```

Build from the command line (requires `sudo xcode-select -s /Applications/Xcode.app/Contents/Developer` to be set):
```
xcodebuild -project TicTacToe.xcodeproj -scheme TicTacToe -destination 'platform=iOS Simulator,name=iPhone 17,OS=26.2' build
```

There are no automated tests at this time.

## Project Structure

The Xcode project is generated by `xcodegen` from `project.yml` — **never edit `TicTacToe.xcodeproj` directly**. All source files live under `TicTacToe/Sources/` and are picked up automatically; no need to register new files in `project.yml`. Plist keys (bundle ID, orientations, AdMob app ID) are defined in `project.yml` under `info.properties` and written to `TicTacToe/Supporting/Info.plist` on each `xcodegen generate` run — edit them in `project.yml`, not the plist directly.

## Architecture

All game logic lives in `GameModel` (a single `ObservableObject`). Views are read-only consumers — they call `game.tap(cell:)`, `game.reset()`, or `game.resetAll()` and re-render from published state. There is no navigation stack; the entire app is a single `ContentView` screen.

**Data flow:**
- `GameModel` owns `board: [Player?]` (9-element flat array, row-major), `currentPlayer`, `state: GameState`, `winningCells: Set<Int>`, and both scores.
- `tap(cell:)` is the only write path. It places the move, checks win patterns, advances the turn, and updates `state`.
- `reset()` starts a new round (alternates first player). `resetAll()` resets scores too.
- Views observe `game` via `@ObservedObject` / `@StateObject` and read directly from its published properties.

**Colors** are defined as `Color` extensions in `Theme.swift` (`.xColor`, `.oColor`, `.winHighlight`) — use these everywhere rather than inline color literals.

## Ads

AdMob is fully wired up and production-ready:
- `AdBannerView.swift` is a `UIViewRepresentable` wrapping `GADBannerView`. It sets `rootViewController` and calls `load()` in `updateUIView` (not `makeUIView`) so the view hierarchy is ready first.
- Real App ID (`ca-app-pub-6670070985256673~9202123662`) is in `project.yml` under `GADApplicationIdentifier`.
- Real Ad Unit ID (`ca-app-pub-6670070985256673/2660766046`) is in `AdBannerView.swift`.
- COPPA child-directed treatment and SDK start are called in `TicTacToeApp.init()`.
- SKAdNetwork identifiers (50 entries) are in `project.yml` under `SKAdNetworkItems`.
- To test on the simulator, temporarily swap the ad unit ID to `ca-app-pub-3940256099942544/2934735716` (Google's test ID) — real ads don't serve on the simulator.

## Key Constraints

- **Portrait only** — locked in `Info.plist`. Do not add landscape support.
- **iOS 16+ / SwiftUI only** — no UIKit views except the AdMob banner wrapper.
- **Bundle ID is hardcoded** as `com.nateguys.tictactoe` in both `project.yml` and `Info.plist`. xcodegen overwrites the plist on each run, so the hardcoded value there is intentional (variable expansion was unreliable in Xcode 26).
